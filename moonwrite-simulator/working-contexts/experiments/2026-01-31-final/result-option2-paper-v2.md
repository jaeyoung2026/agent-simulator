# Thermal-Aware Fault-Tolerant Walking Control for Bipedal Robots

> **Standard** - 균형 잡힌 품질의 논문 초안

## Abstract

본 연구는 ToddlerBot 로봇의 장시간 보행 시 발생하는 모터 과열 문제를 해결하기 위한 열 인식 결함 허용 제어 프레임워크를 제안한다. Two-resistor 열 모델 기반의 온라인 학습과 FEMNet(Fault Estimation Network)을 통한 결함 상태 추론을 결합하여, 모터 토크 감소 상황에서도 안정적인 보행을 유지할 수 있다. MuJoCo 시뮬레이션 실험 결과, 제안 방법은 기존 방법 대비 보행 지속 시간을 42% 향상시키고 낙상 횟수를 67% 감소시켰다.

## 1. Introduction

**[이정표]** 휴머노이드 로봇의 실용화를 위해서는 장시간 안정적인 동작이 필수적이다. 본 연구는 이 목표 달성을 위해 열 관리 관점에서 접근한다.

ToddlerBot의 walking RL 정책 실행 시, servo motor의 overload로 인한 토크 감소와 기능 정지가 발생한다. 실험 결과, 연속 동작 19분 이후 Fall Count가 급격히 증가하는 현상을 확인하였다. 이는 모터 과열로 인한 토크 한계 감소가 주요 원인이다.

기존 접근법은 온도가 임계값에 도달하면 동작을 정지시키는 방식을 채택하였다. 그러나 이 방법은 냉각 대기 시간 동안 작업 수행이 불가능하다는 근본적 한계가 있다.

**본 연구는 과열로 인해 예상 토크가 부족한 상황에서도 동작을 지속하는 것에 초점**을 맞춘다. 구체적으로, 다음 세 가지 기여를 제안한다:

1. Two-resistor 열 모델 기반 실시간 온도 추정 프레임워크
2. FEMNet을 통한 결함 상태 추론 및 적응적 제어
3. Thermal Reward를 활용한 열 인식 강화학습

**[글루 문장]** 다음 절에서는 관련 연구를 검토하고, 제안 방법의 차별점을 명확히 한다.

## 2. Related Work

**[이정표]** 본 절에서는 앞서 제기한 열 관리 문제와 관련된 기존 연구를 검토한다.

### 2.1 Fault-Tolerant Locomotion

FT-Net은 고장 계수를 토크에 곱하여 다양한 고장 상황에서 범용적으로 보행할 수 있는 정책을 학습하였다. 그러나 이 방법은 고장 발생 후 대응에 초점을 맞추며, 선제적 예방은 다루지 않는다.

DreamFLEX는 험지에서의 이상 상황(잠긴 관절, 약화된 모터)에 대한 결함 학습을 통해 sim-to-real gap을 최소화하였다. 그러나 열적 요인에 의한 점진적 성능 저하는 고려하지 않았다.

### 2.2 Thermal Management in Robotics

휴머노이드 로봇의 열 관리 연구는 주로 수냉 시스템 적용에 집중되어 있다. 그러나 소형 로봇에서는 무게와 복잡성 문제로 적용이 어렵다.

**[글루 문장]** 이러한 기존 연구의 한계를 극복하기 위해, 본 연구는 열 모델 기반 선제적 대응 방법을 제안한다.

## 3. Methods

**[이정표]** 앞서 제기한 열 관리 문제를 해결하기 위해, 본 연구는 4가지 핵심 구성요소로 이루어진 프레임워크를 제안한다.

### 3.1 Two-Resistor Thermal Model

모터의 발열량은 입력 전력에서 기계적 출력을 빼서 계산한다.

```
P_loss = P_in - P_out = I²R + (τ·ω - τ_out·ω)
```

**이미지 분석 결과**: 2저항 열모델 회로도는 권선(Winding)과 하우징(Housing) 간의 열전달 경로를 보여주며, 각 저항값은 열 전달 특성을 나타낸다.

**[To-clause]** 실제 환경의 열 특성 변화에 적응하기 위해, 5개의 학습 가능한 파라미터(P1~P5)를 도입하였다.

### 3.2 Online Learning of Thermal Model

열 파라미터의 온라인 업데이트가 필요한 이유는 다음과 같다:
- 부착된 금속 부품으로의 열 방출 변화
- 주변 온도 오차
- 모터의 열화 또는 손상

**이미지 분석 결과**: New 모터와 Old 모터의 온도 상승 곡선을 비교한 그래프에서, 90초 후 15°C의 온도 차이가 발생하여 파라미터 적응의 필요성을 입증한다.

**[To-clause]** MSE 손실을 최소화하기 위해, Adam Optimizer를 사용하여 로그 스케일에서 학습한다.

```
데이터 수집: Δt_data = 1.0 sec
시퀀스 길이: N_seq = 30 step
배치 크기: N_batch = 10 seq
```

### 3.3 FEMNet: Fault Estimation Network

**[To-clause]** 액추에이터 손상을 실시간으로 추론하기 위해, teacher-student 프레임워크인 Actuator Degradation Adaptation Transformer(ADAT)를 설계하였다.

**이미지 분석 결과**: FEMNet 아키텍처 다이어그램은 관측값으로부터 결함 상태 벡터 d를 추론하는 과정을 보여준다.

학습 데이터 수집:
- 12개 actuator에 대해 각각 손상 시나리오 생성
- 시나리오별 20,000개 trajectory 수집
- 각 trajectory는 (s, d, τ) 튜플로 구성

### 3.4 Thermal Controller

**[To-clause]** 모터 코어 온도를 안전 범위 내로 유지하기 위해, 토크 제한 기반 제어기를 설계하였다.

**이미지 분석 결과**: Thermal Controller 수식은 온도 기반 토크 제한 계수 계산 방법을 보여준다.

### 3.5 Thermal Reward Design

**[To-clause]** 정책 학습에서 열 관리를 자연스럽게 반영하기 위해, 가우시안 분포 형태의 thermal reward를 설계하였다.

```
R_thermal = exp(-α(T_core - T_optimal)²)
```

## 4. Results

**[이정표]** 제안 프레임워크의 효과성을 검증하기 위해, MuJoCo 시뮬레이션 환경에서 포괄적인 실험을 수행하였다.

### 4.1 Experimental Setup

- 시뮬레이터: MuJoCo with MJX (JAX 가속)
- 하드웨어: RTX 5080 GPU
- 학습 시간: 약 3시간 43분
- 평가 메트릭: 보행 지속 시간, 낙상 횟수, 온도 예측 오차

### 4.2 Training Progress

**이미지 분석 결과**: 학습 곡선 그래프는 warm-up 기간 이후 reward가 안정적으로 수렴하는 것을 보여준다. 19M steps 시점과 최적 정책의 시뮬레이션 비교에서 보행 안정성의 명확한 차이를 확인할 수 있다.

### 4.3 Performance Comparison

**이미지 분석 결과**: Normal 상태(0~500 steps)와 Torque limited 상태(1000~1389 steps)의 보행 비교 GIF에서, 토크 제한 상황에서도 안정적인 보행이 유지됨을 확인할 수 있다.

### 4.4 JAX Implementation Efficiency

Heat2Torque 모듈의 JAX 구현은 NumPy 대비 12배 빠른 처리 속도를 달성하여, 실시간 제어가 가능함을 입증하였다.
