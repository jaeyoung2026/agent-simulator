# 실험 2: LLM 기반 분류/처리 아이디어 비교

> 첫 번째 실험의 피드백 반영:
> - 기술적 구현(임베딩 등)보다 **분류/처리 아이디어**에 집중
> - LLM을 적극 활용
> - 실제 슬라이드 데이터 사용 (275개)

## 1. 실험 데이터

```

## 2. 비교할 아이디어

### 아이디어 A: 관점(Perspective) 변경

동일한 슬라이드를 **다른 관점**에서 분석하면 어떤 차이가 있는가?

| 관점 | 질문 | 분류 기준 |
|-----|------|----------|
| **논문 구조** | "이것이 논문 어디에 들어가나?" | IMRaD (Intro, Method, Result, Discussion) |
| **연구 스토리** | "연구가 어떻게 진행되었나?" | 시간순 (계획→실행→발견→해석) |
| **논증 흐름** | "무엇을 주장하고 어떻게 뒷받침하나?" | 논증 (주장, 근거, 반론, 한계) |
| **문제-해결** | "어떤 문제를 어떻게 해결했나?" | Problem-Solution (문제→시도→해결→교훈) |

**가설**: 연구 발표 슬라이드는 "연구 스토리" 또는 "문제-해결" 관점이 더 자연스러울 수 있음

---

### 아이디어 B: 추출 단위(Granularity) 변경

슬라이드에서 **어느 수준**으로 의미를 추출할 것인가?

| 수준 | 설명 | 예상 결과 |
|-----|------|----------|
| **슬라이드 단위** | 슬라이드 1개 = 의미 1개 | 275개 (슬라이드 수) |
| **주장 단위** | 하나의 주장 = 의미 1개 | 300~400개 (예상) |
| **문장 단위** | 문장 1개 = 의미 1개 | 1000개+ |
| **토픽 단위** | 여러 슬라이드 = 토픽 1개 | 50~100개 (예상) |

**가설**: "주장 단위"가 논문 작성에 가장 적합할 것

---

### 아이디어 C: 처리 순서(Order) 변경

어떤 순서로 분석하면 품질이 높은가?

| 순서 | 방식 | 장단점 |
|-----|------|--------|
| **개별 → 통합** | 슬라이드별 분석 후 합치기 | 병렬 가능, 맥락 손실 |
| **전체 → 개별** | 전체 요약 먼저, 개별 분석 | 맥락 유지, 순차적 |
| **대표 → 확장** | 핵심 슬라이드 먼저, 나머지 배치 | 효율적, 핵심 선정 어려움 |
| **계층적** | 파일→섹션→슬라이드 순으로 | 구조화, 복잡함 |

**가설**: "전체 → 개별" 또는 "대표 → 확장"이 일관성 높을 것

---

### 아이디어 D: 분류 체계(Schema) 변경

어떤 분류 체계가 연구 발표에 적합한가?

| 체계 | 카테고리 | 특징 |
|-----|---------|------|
| **현재 (5단계)** | 배경, 방법, 결과, 해석, 기여 | 논문 중심 |
| **IMRaD** | Intro, Method, Result, Discussion | 학술 표준 |
| **연구 단계** | 동기, 설계, 구현, 실험, 분석, 계획 | 진행 중심 |
| **Weekly Report** | 완료, 진행중, 계획, 이슈, 요청 | 보고서 중심 |

**가설**: "Weekly Report" 체계가 실제 데이터와 더 맞을 수 있음

---

### 아이디어 E: 프롬프트 전략(Strategy) 변경

| 전략 | 설명 | 예상 효과 |
|-----|------|----------|
| **Direct** | 바로 분류 요청 | 빠름, 일관성 낮음 |
| **Chain-of-Thought** | 단계별 추론 후 분류 | 느림, 근거 명확 |
| **Self-Critique** | 분류 후 자기 검토 | 정확도 향상 |
| **Comparative** | 다른 슬라이드와 비교하며 분류 | 일관성 향상 |

---

### 아이디어 F: 이미지 활용(Multimodal) 전략

슬라이드당 평균 1.6개 이미지 (440개/275슬라이드). 이미지를 어떻게 활용할 것인가?

| 전략 | 설명 | 예상 효과 |
|-----|------|----------|
| **텍스트만** | 이미지 무시, 텍스트만 분석 | 빠름, 정보 손실 |
| **이미지 설명 추가** | LLM이 이미지 설명 생성 → 텍스트와 결합 | 풍부한 맥락 |
| **이미지 우선** | 이미지로 먼저 분류, 텍스트로 보완 | 시각 정보 중심 |
| **통합 분석** | 텍스트 + 이미지를 동시에 멀티모달 분석 | 최고 품질, 비용 높음 |

**가설**: 그래프/다이어그램이 많은 슬라이드는 이미지 분석이 분류에 결정적일 수 있음

### 이미지 유형별 역할

| 이미지 유형 | 역할 | 분류 영향 |
|------------|------|----------|
| **그래프/차트** | 결과 시각화 | → Result 분류에 강한 신호 |
| **아키텍처 다이어그램** | 방법론 설명 | → Method 분류에 강한 신호 |
| **스크린샷** | 구현 증거 | → Result/Method |
| **플로우차트** | 프로세스 설명 | → Method |
| **비교 표** | 결과 비교 | → Result/Discussion |
| **로고/장식** | 의미 없음 | 필터링 대상 |

## 3. 실험 설계

### 3.1 샘플 선정

```
전체 275개 중 대표 샘플 20개 선정:
- 초기 파일 (WR 07_01)에서 5개
- 중간 파일 (WR 09_23)에서 10개
- 최근 파일 (WR 12_23)에서 5개

선정 기준: 내용이 풍부한 슬라이드 (표지, 목차 제외)
```

### 3.2 비교 실험

**실험 1: 관점 비교** (아이디어 A)
- 동일 샘플 20개를 4가지 관점으로 분석
- 어떤 관점이 더 자연스러운 분류를 하는지 비교

**실험 2: 분류 체계 비교** (아이디어 D)
- 동일 샘플을 4가지 분류 체계로 분류
- 어떤 체계가 빈 카테고리 없이 균형 있게 분류되는지 확인

**실험 3: 처리 순서 비교** (아이디어 C)
- "개별 → 통합" vs "전체 → 개별" 비교
- 분류 일관성과 품질 비교

### 3.3 평가 기준

| 기준 | 측정 방법 |
|-----|----------|
| **분류 균형** | 각 카테고리에 할당된 비율 |
| **분류 일관성** | 유사 내용이 같은 카테고리로 분류되는지 |
| **자연스러움** | 분류 결과가 직관적으로 맞는지 (수동 검토) |
| **누락 없음** | 모든 슬라이드가 적절히 분류되었는지 |

---

## 4. 예상 인사이트

1. **연구 발표 ≠ 논문**
   - 논문 구조(IMRaD)가 아닌 "진행 보고" 구조가 더 맞을 수 있음

2. **시간 축이 중요**
   - 같은 주제도 시간에 따라 "계획→진행→완료"로 변화
   - 최신 정보 우선 원칙이 중요

3. **맥락이 분류를 결정**
   - 개별 슬라이드만 보면 오분류
   - 전체 맥락(파일 제목, 앞뒤 슬라이드)이 필요

---

1. 샘플 20개 선정 및 추출

4. 실험 3: 처리 순서 비교 (2가지)

```
