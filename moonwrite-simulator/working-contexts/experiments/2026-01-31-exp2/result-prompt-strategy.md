# 프롬프트 전략 비교 실험 결과

## 실험 개요

**실험 일자**: 2026-01-31
**목적**: 동일한 슬라이드 데이터를 3가지 프롬프트 전략으로 분류하여 각 전략의 특성 비교

### 분류 카테고리 (연구 스토리)
| 카테고리 | 설명 |
|---------|------|
| Planning | 연구 계획, 목표 설정, 아이디어 제시 |
| Setup | 환경 구축, 도구 설정, 데이터 준비 |
| Execution | 실험 수행, 알고리즘 구현, 방법론 적용 |
| Finding | 실험 결과, 데이터 분석, 측정값 |
| Interpretation | 결과 해석, 한계점 분석, 의미 부여 |

### 3가지 프롬프트 전략

| 전략 | 특징 | 프롬프트 구조 |
|------|------|--------------|
| **Direct** | 단일 지시문으로 바로 분류 | "이 슬라이드를 [카테고리 목록] 중 하나로 분류하세요" |
| **Chain-of-Thought (CoT)** | 단계적 추론 과정 | 1.요약 → 2.역할 분석 → 3.분류 선택 |
| **Self-Critique** | 분류 후 자기 검토 | 1.초기 분류 → 2.대안 검토 → 3.최종 확정 |

- **근거**: 모터 열 관리 시스템 구축에 관한 내용

1. **요약**: 휴머노이드 로봇의 전기 모터 열 제어를 위해 수냉 시스템을 적용하고, two-resister 열 모델로 온도를 계산하는 방법 제안

2. **검토**: Setup은 환경/도구 설정에 가까움. 이 슬라이드는 "방법을 제안"하고 있어 연구 방법론 실행에 더 가까움
3. **최종 분류**: **Execution**

## 분석 결과

### 1. 분류 일치율

| 비교 항목 | 일치 수 | 일치율 |
|----------|---------|--------|
| 3전략 전체 일치 | 4/5 | **80%** |
| Direct vs CoT | 4/5 | 80% |
| Direct vs Self-Critique | 4/5 | 80% |
| CoT vs Self-Critique | 5/5 | **100%** |

### 2. 분류 근거의 구체성

| 전략 | 근거 구체성 | 특징 |
|------|------------|------|
| **Direct** | **낮음** | 단순 키워드 매칭 (제목, 주요 단어) |
| **CoT** | **높음** | 단계적 분석을 통해 맥락 이해 → 정확한 역할 파악 |
| **Self-Critique** | **매우 높음** | 대안 검토를 통해 분류 근거 강화, 경계 사례 구분 |

### 3. 전략별 강점/약점

#### Direct (직접 분류)
- **강점**: 빠른 처리, 명확한 키워드가 있을 때 효과적
- **약점**: 맥락 파악 부족, 표면적 단어에 의존
- **적합 사례**: 명확한 제목/키워드가 있는 슬라이드

#### Chain-of-Thought (단계적 추론)
- **강점**: 내용의 역할을 깊이 분석, 맥락 기반 판단
- **약점**: 처리 시간 증가
- **적합 사례**: 애매한 경계 사례, 복합적 내용

#### Self-Critique (자기 검토)
- **강점**: 분류 신뢰도 향상, 경계 사례에서 정확도 증가
- **약점**: 가장 긴 처리 시간, 단순 사례에는 과도할 수 있음
- **적합 사례**: 고정밀 분류가 필요한 경우

### 4. 불일치 사례 심층 분석

**WR 8_5 슬라이드 7 (열 제어 방법)**

| 전략 | 분류 | 핵심 판단 근거 |
|------|------|---------------|
| Direct | Setup | "수냉 적용", "모델 사용" → 시스템 구축 |
| CoT | Execution | "방법 제안" → 연구 방법론의 핵심 구현 |
| Self-Critique | Execution | Setup vs Execution 비교 후 → 단순 설정이 아닌 방법론 |

**분석**: Direct 전략은 "수냉 적용", "온도 계산" 등의 표면적 키워드에 기반하여 Setup으로 판단. 반면 CoT와 Self-Critique는 "방법을 제안"이라는 맥락을 파악하여 이것이 연구의 핵심 방법론(Execution)임을 인식.

## 결론 및 권장사항

### 주요 발견

1. **CoT와 Self-Critique의 높은 일치율 (100%)**: 추론 과정을 거치면 분류 결과가 수렴
2. **Direct 전략의 한계**: 키워드 의존으로 "방법 제안" 같은 맥락적 의미 놓칠 수 있음
3. **경계 사례에서의 차이**: Setup vs Execution 구분에서 추론 기반 전략이 우수

### 권장 프롬프트 전략

| 상황 | 권장 전략 | 이유 |
|------|----------|------|
| 대량 처리 (1000+ 슬라이드) | Direct + 샘플링 검증 | 효율성, 명확 사례 빠른 처리 |
| 정밀 분류 필요 | Self-Critique | 최고 신뢰도 |
| 균형잡힌 접근 | **Chain-of-Thought** | 정확도와 효율성의 최적 균형 |

### 하이브리드 전략 제안

```
1단계: Direct로 1차 분류
2단계: 신뢰도 낮은 사례 (경계 사례) 식별
3단계: 경계 사례에만 Self-Critique 적용
```

이 접근법으로 효율성과 정확도를 모두 확보할 수 있음.
