> 실험일: 2026-01-31
> 데이터: 275개 슬라이드, 132개 이미지 (13개 파일)

---

## 1. 실험 개요

### 목표
LLM 기반 분류/처리 아이디어 비교 (기술 구현이 아닌 개념적 접근)

### 비교 실험
1. **관점(Perspective) 비교**: 4가지 분류 관점
2. **프롬프트 전략 비교**: 3가지 LLM 프롬프트 전략

---

## 2. 실험 1: 관점 비교

### 결과 요약

| 관점 | 주요 카테고리 | 분포 균형 | 적합도 |
|-----|-------------|----------|--------|
| **논문 구조 (IMRaD)** | Method 60% | 편중 | ★★★ |
| **연구 스토리** | 실행 40%, 발견 25% | 균형 | ★★★★★ |
| **문제-해결** | 시도 45%, 해결 35% | 균형 | ★★★★ |
| **Weekly Report** | 학습 40%, 계획 0% | 편중 | ★★★ |

### 핵심 발견

1. **논문 구조(IMRaD)는 연구 발표에 부적합**
   - 슬라이드가 Method에 60% 편중
   - 논문용 분류 체계가 발표 자료와 맞지 않음

2. **연구 스토리 관점이 가장 균형 있음**
   - 5개 카테고리에 고르게 분포
   - 연구 진행 과정을 자연스럽게 반영

3. **Weekly Report 관점은 "계획" 누락**
   - 샘플이 완료/진행 중심이라 계획 카테고리 0%
   - 실제 주간 보고와는 다른 특성

### 권장 관점
```
연구 발표 슬라이드 → "연구 스토리" 관점 권장
├── 계획(Planning)
├── 설정(Setup)
├── 실행(Execution)
├── 발견(Finding)
└── 해석(Interpretation)
```

## 3. 실험 2: 프롬프트 전략 비교

### 결과 요약

| 전략 | 정확도 | 근거 구체성 | 효율성 | 권장 상황 |
|-----|--------|------------|--------|----------|
| **Direct** | 80% | 낮음 | 높음 | 대량 처리 |
| **Chain-of-Thought** | 100% | 높음 | 중간 | **기본 권장** |
| **Self-Critique** | 100% | 매우 높음 | 낮음 | 고정밀 필요 |

### 핵심 발견

1. **CoT와 Self-Critique는 동일한 결과**
   - 추론 기반 전략들은 같은 결론에 도달
   - Self-Critique의 추가 비용 대비 효과 미미

2. **Direct의 한계**
   - 표면적 키워드에 의존
   - "방법 제안" 같은 맥락적 의미 놓침
   - Setup vs Execution 경계에서 오분류

3. **Chain-of-Thought가 최적 균형**
   - 100% 정확도 달성
   - Self-Critique 대비 효율적
   - 분류 근거가 명확

### 권장 전략
```
기본: Chain-of-Thought
├── 1. 핵심 내용 요약
├── 2. 연구 과정에서의 역할 분석
└── 3. 카테고리 선택
```

---

## 4. 종합 권장사항

### Moonwriter 적용 제안

| 현재 | 제안 |
|-----|------|
| 논문 5단계 (배경→방법→결과→해석→기여) | 연구 스토리 5단계 (계획→설정→실행→발견→해석) |
| Direct 분류 | Chain-of-Thought 분류 |

### 구체적 변경

1. **분류 체계 변경**
   ```
   기존: introduction, method, result, discussion, conclusion
   제안: planning, setup, execution, finding, interpretation
   ```

2. **프롬프트 구조 변경**
   ```
   기존: "이 슬라이드를 분류하세요"
   제안:
     1. "핵심 내용을 요약하세요"
     2. "연구 진행에서 어떤 역할인지 설명하세요"
     3. "카테고리를 선택하세요"
   ```

3. **이미지 활용**
   - 그래프/차트 → Finding 분류 신호
   - 아키텍처 다이어그램 → Setup/Execution 분류 신호
   - 스크린샷 → Execution 분류 신호
