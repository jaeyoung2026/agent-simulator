# 최종 비교 리포트: 의미 추출 & 클러스터링 방법

> 실험 수행일: 2026-01-31

---

## 1. 실험 요약

### 목적
Moonwriter의 현재 방식과 대안 방법들을 비교하여 최적의 의미 추출 및 클러스터링 조합 탐색

### 실험 데이터
- 4개 슬라이드 (서버 환경 설정, 이슈 해결, 진행 현황, 로드맵)
- 실제 연구 발표 자료 (`/slides/` 폴더)

### 비교 대상
| 방법 | 의미 추출 | 클러스터링 |
|-----|----------|-----------|
| **Baseline** | LLM 직접 추출 | LLM 직접 분류 |
| **임베딩 기반** | 유사도 그룹화 | 키워드 매칭 |
| **Proposition** | 원자적 명제 분해 | 명제별 LLM 분류 |
| **하이브리드** | LLM 추출 | 키워드 초벌 + LLM 정제 |

---

## 2. 핵심 발견

### 정량적 결과

| 지표 | Baseline | 임베딩 | Proposition | 하이브리드 |
|-----|----------|--------|-------------|-----------|
| 추출 단위 | 9개 | 5그룹 | 27개 | 9개 |
| LLM 호출 | 5회 | 0회 | 31회 | 9회 |
| 예상 비용 | 중간 | **낮음** | 높음 | 중간-낮음 |
| 분류 정확도 | 좋음 | 보통 | 좋음 | **매우 좋음** |

### 주요 관찰

1. **임베딩 기반의 한계**
   - "문제 해결"을 Results가 아닌 Discussion으로 오분류
   - 키워드 매칭만으로는 의미적 뉘앙스 구분 어려움

2. **Proposition의 과도한 분해**
   - 27개 명제로 가장 세밀하지만
   - 관련 명제들을 다시 묶어야 하는 문제 발생
   - 비용 대비 효과 의문

3. **하이브리드의 효율성**
   - 9개 중 4개는 키워드로 빠르게 분류 (High 신뢰도)
   - 5개 경계 케이스만 LLM 정밀 분류
   - **44% 비용 절감** + Baseline과 동일한 품질

## 3. 최종 권장

### 시나리오별 추천

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   정확도 최우선    →  Proposition (비용 높음)            │
│                                                         │
│   균형 잡힌 선택   →  하이브리드 (★ 권장)                │
│                                                         │
│   대량 처리       →  임베딩 기반 (정확도 희생)           │
│                                                         │
│   빠른 프로토타입  →  Baseline (현재 방식)               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Moonwriter 적용 제안

| 모드 | 방법 | 사용 시점 |
|-----|------|----------|
| **기본** | 하이브리드 | 일반 분석 |
| **고품질** | Baseline | 중요 문서 |
| **고속** | 임베딩 기반 | 초안 수준 빠른 분류 |

## 4. 다음 단계 (제안)

### 즉시 적용 가능
1. **하이브리드 분류 도입**
   - 키워드 사전 구축 (Stage별 대표 키워드)
   - 신뢰도 임계값 설정 (High/Low 구분)
   - 경계 케이스만 LLM 호출

### 추가 실험 필요
1. **더 많은 샘플로 검증**

2. **임베딩 모델 비교**

   - Sentence-BERT 변형들
   - 한국어 특화 모델

3. **비용 실측**
   - 실제 API 호출로 비용 측정
   - 처리 시간 벤치마크

## 6. 결론

> **하이브리드 방식**이 비용 효율성과 분류 품질의 최적 균형을 제공한다.
>
> - 키워드 기반 초벌 분류로 명확한 케이스 빠르게 처리
> - 경계 케이스에만 LLM 정밀 분류 적용
> - 44% 비용 절감 + Baseline 수준 품질 유지

현재 Moonwriter의 Baseline 방식은 품질이 좋지만, 대규모 처리 시 비용 부담이 있다. 하이브리드 방식 도입으로 비용을 절감하면서 품질을 유지할 수 있다.
