# 에이전트 활용 가이드: 의미 분리 & 클러스터링 실험

## 1. 핵심 원칙

### 에이전트는 자율적으로 판단하고 실행한다

| 당신의 역할 | 에이전트의 역할 |
|------------|---------------|
| 목표 제시 | 계획, 조사, 구현, 실행 |
| 리포트 검토 | 진행 상황 보고 |
| 방향 수정 (필요시) | 자율적 판단 및 반복 |

### 자율 실행 원칙

```
✓ "의미 분리와 클러스터링 방법을 조사하고 비교 실험해줘"
  → 에이전트가 스스로 단계를 나누고, 조사하고, 설계하고, 실행
  → 각 단계마다 진행 상황 리포트
```

**목표 + 신뢰 = 자율 실행**

---

## 2. 리포트 형식

에이전트는 각 단계에서 다음 형식으로 리포트한다:

### 단계 시작 리포트

```markdown
## [단계명] 시작

**목표**: 이번 단계에서 달성할 것
**접근 방식**: 어떻게 진행할 것인지
**예상 산출물**: 무엇이 나올 것인지
```

### 단계 완료 리포트

```markdown
## [단계명] 완료

**수행한 작업**: 무엇을 했는지
**산출물**: 생성된 파일/결과
**주요 발견**: 중요한 인사이트
**다음 단계**: 이어서 할 작업
```

### 의사결정 리포트 (선택적)

```markdown
## 의사결정 필요

**상황**: 현재 상태
**선택지**: A, B, C
**내 판단**: B를 선택하려 함
**이유**: 왜 B가 좋은지

→ 이대로 진행해도 될까요? (또는 그냥 진행)
```

## 3. 실험 사이클 구조

```
┌─────────────────────────────────────────────────────────┐
│                    한 사이클                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 조사 (Research)                                     │
│     └─ 의미 추출 방법 + 클러스터링 방법 조사              │
│     └─ 산출물: research-*.md                            │
│                                                         │
│  2. 설계 (Design)                                       │
│     └─ Moonwriter 맥락에서 적용할 방법 선정              │
│     └─ 산출물: experiment-design.md                     │
│                                                         │
│  3. 구현 (Implementation)                               │
│     └─ 선정된 방법들 프로토타입 구현                     │
│     └─ 산출물: lib/experiments/*.ts 또는 프롬프트        │
│                                                         │
│  4. 실험 (Experiment)                                   │
│     └─ 샘플 데이터로 각 방법 실행                        │
│     └─ 산출물: experiment-results.md                    │
│                                                         │
│  5. 비교 (Comparison)                                   │
│     └─ 결과 비교 및 인사이트 도출                        │
│     └─ 산출물: comparison-report.md                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```

→ 13개의 실제 연구 발표 슬라이드. 실험에 활용 가능.

### 서브에이전트 활용

| 에이전트 | 용도 |
|---------|------|
| **Explore** | 코드베이스 탐색, 파일 검색 |
| **Bash** | 명령어 실행, 파일 처리 |
| **general-purpose** | 복잡한 조사, 다단계 작업 |

→ 병렬 작업이 필요하면 서브에이전트를 적극 활용

### 인터넷 조사

- **WebSearch**: 최신 방법론, 논문, 도구 검색
- **WebFetch**: 특정 페이지 내용 조회

→ 조사 단계에서 인터넷 검색 적극 활용

```

## 7. 에이전트 자율 판단 영역

### 에이전트가 스스로 결정할 수 있는 것

- 조사할 방법의 범위와 깊이
- 실험에 사용할 샘플 데이터 선정/생성
- 구현 방식 (코드 vs 프롬프트)
- 비교 기준 설정
- 파일 구조 및 명명

### 사용자에게 물어봐야 할 것

- 실험 방향의 근본적 변경
- 예상치 못한 큰 비용/시간 소요
- 명확한 선택지가 있고 사용자 선호가 중요할 때
